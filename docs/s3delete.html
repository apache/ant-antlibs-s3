s3:delete

Description

Delete the objects represented by a collection of s3:object resources.

Parameters

Attribute Description Required
clientrefid Ant reference ID to an s3:client defined elsewhere No
refid Ant reference ID to a resource collection to delete No
as object precision: object|version No, default object
blocksize Size of delete batches when operating in batch mode No, default 1000
verbose Log activity verbosely No, default false

Parameters specified as nested elements

s3:client

s3:client to use for deletion. If a client is specified either in this manner or by use of @clientrefid, the delete operation is put into batch mode. If the collection of s3:object resources come from an s3:objects collection, and therefore already have a reference to the client that discovered them, this task does not need a client reference and each object will be requested to delete itself.

resource collection

Any number of nested resource collections may be specified; however resources other than s3:object are ignored. Further, if @as is object an s3:object that is a delete marker will also be ignored, as it indicates the object has already been deleted, so far as it goes.
