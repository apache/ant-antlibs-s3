Introduction

This is a library of Ant type definitions that support working with Amazon Web Services' Simple Storage Service AKA S3.

Requirements

The current version requires Ant 1.10.7 or later.

Where is it?

The source code for the library lives in the ant-antlibs-s3 git repository - https://gitbox.apache.org/repos/asf/ant-antlibs-s3.git.

Installation

If you are building this from sources, run the antlib target and you'll get a file ant-s3.jar.

The defined types are named so as to encourage their use via namespace declaration. Place ant-s3.jar into a directory and use ant -lib DIR-CONTAINING-THE-JAR or copy it into ANT_HOME/lib - and then in your build file, simply declare the namespace on the project tag: 

        <project xmlns:s3="antlib:org.apache.ant.s3">
...and the types defined by this library will automatically be available in the s3 namespace. It is planned to publish the artifact with dependency information such that tools like Apache Ivy or Apache Maven can retrieve all required dependencies.

Supplied Tasks

  • s3:put - Put an Ant ResourceCollection into an S3 bucket.
  • s3:delete - Delete objects from an S3 bucket.

Supplied Types

  • s3:client - Represents an instance of the AWS SDK's 
    AmazonS3
    client interface.
  • s3:object - Ant Resource implementation representing a single S3 object.
  • s3:objects - Ant ResourceCollection collecting the objects located at a particular bucket and prefix.

Custom resource comparators/selectors

A number of types are provided that implement both interfaces for use with Ant's sort and restrict resource collections. These are situated in a secondary Antlib namespace org.apache.ant.s3cs.
